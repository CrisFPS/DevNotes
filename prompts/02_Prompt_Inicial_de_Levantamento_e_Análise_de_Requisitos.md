# Prompt Inicial de Levantamento e Análise de Requisitos

Atue como analista de requisitos de software.

Quero levantar os requisitos iniciais de um pequeno sistema chamado DevNotes Local.

O sistema será uma aplicação web local para eu guardar e pesquisar conteúdos técnicos, como snippets, SQLs, anotações, scripts, regras de negócio e arquivos de sistemas legados.

Quero que a aplicação seja simples, apenas para estudo e uso local.

Funcionalidades que imagino:
- cadastrar uma anotação técnica;
- editar;
- excluir;
- pesquisar;
- fazer upload de um arquivo;
- classificar por linguagem, sistema, domínio, tags e categoria;
- marcar algo como regra de negócio;
- visualizar o conteúdo preservando a formatação;
- destacar código com Highlight.js.

Tecnologias que pretendo usar:
- Python 3.11;
- FastAPI;
- SQLite;
- SQLAlchemy;
- SQLite FTS5;
- Jinja2;
- HTML e CSS simples;
- pytest.

Não quero neste MVP:
- login;
- vários usuários;
- sistema em produção;
- microsserviços;
- frontend sofisticado;
- backup automático.

Quero que você me ajude a transformar essa ideia em requisitos iniciais.

Gere uma resposta em Markdown contendo:

1. Objetivo do sistema.
2. Problema que o sistema resolve.
3. Lista de requisitos funcionais.
4. Lista de requisitos não funcionais.
5. Regras de negócio iniciais.
6. Restrições do MVP.
7. Dúvidas ou pontos que precisam ser esclarecidos.
8. Sugestão de próximos passos.

Não implemente código.
Não crie arquitetura detalhada ainda.
Não crie tarefas técnicas ainda.
Apenas faça uma primeira versão simples do levantamento de requisitos.
